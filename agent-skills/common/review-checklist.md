# コードレビューチェックリスト

> **使い方**: PR レビュー時にこのチェックリストを参照する。
> Claude Code では `/review-pr` コマンドがこのテンプレートを使用する。

---

## 機能・ロジック

- [ ] 要件・仕様を満たしているか
- [ ] エッジケース・異常系が考慮されているか
- [ ] 既存機能への影響・デグレードがないか

## コード品質

- [ ] 命名が意図を明確に表しているか
- [ ] 不要なコード・デバッグ用コードが残っていないか
- [ ] 重複ロジックが排除されているか（DRY 原則）
- [ ] 関数・モジュールが適切な粒度に分割されているか

## セキュリティ

- [ ] SQL インジェクション・XSS 等の脆弱性がないか
- [ ] シークレット・認証情報がハードコードされていないか
- [ ] 入力値の検証が適切か

## テスト

- [ ] テストが追加・更新されているか
- [ ] テストケースが主要なシナリオをカバーしているか

## ドキュメント

- [ ] 必要なコメントが追加されているか
- [ ] README / CHANGELOG の更新が必要な場合は更新されているか

## 運用

- [ ] ログ・エラーハンドリングが適切か
- [ ] パフォーマンスへの影響が考慮されているか
- [ ] マイグレーション・デプロイ手順に問題がないか
